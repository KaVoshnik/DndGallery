# Гайд: как добавить предмет в Архив Шмотья

Сайт читает все предметы из одного файла — **`items.json`**. Чтобы добавить
новый предмет, не нужно ничего программировать: правишь файл прямо на сайте
GitHub, сохраняешь — и через минуту-другую изменения сами появятся на странице
(GitHub Pages пересобирает сайт автоматически после каждого сохранения).

---

## Шаг 1. Открой items.json на GitHub

1. Зайди в репозиторий проекта на github.com.
2. Найди в списке файлов `items.json` и кликни на него.
3. Справа сверху нажми на иконку карандаша **"Edit this file"** (или клавишу `e`).

Откроется текстовый редактор прямо в браузере — выглядит как блокнот с
подсветкой кода.

---

## Шаг 2. Пойми структуру файла

Файл — это список карточек в фигурных скобках `{ }`, разделённых запятыми,
всё это обёрнуто в квадратные скобки `[ ]`. Один предмет — одна карточка:

```json
{
  "id": "krasnyy-vihr",
  "name": "Красный Вихрь",
  "rarity": "rare",
  "type": "weapon",
  "subtype": "Длинный меч",
  "image": "images/krasnyy-vihr.jpg",
  "description": "Длинный меч с красным лезвием..."
}
```

### Что значит каждое поле

| Поле | Что писать | Обязательно? |
|---|---|---|
| `id` | Уникальный код предмета, **латиницей**, в стиле `slovo-cherez-defis`. Используется только внутри сайта, на странице не отображается. Не должно повторяться у двух предметов. | Да |
| `name` | Название предмета — то, что увидит игрок. | Да |
| `rarity` | Редкость. Только одно из значений из таблицы ниже. | Да |
| `type` | Тип предмета. Только одно из значений из таблицы ниже. | Да |
| `subtype` | Подпись помельче — конкретный вид предмета ("Длинный меч", "Брошь", "Кольцо"). Можно оставить пустой строкой `""`, если не нужно. | Нет |
| `image` | Путь к картинке предмета (см. Шаг 3). Если картинки нет — оставь `""`, сайт сам подставит золотую иконку-заглушку. | Нет |
| `description` | Полный текст предмета — то, что раскрывается при клике на карточку. | Да |

### Допустимые значения `rarity`

```
common       — Обычный
uncommon     — Необычный
rare         — Редкий
very-rare    — Очень редкий
legendary    — Легендарный
artifact     — Артефакт
```

### Допустимые значения `type`

```
weapon     — Оружие
armor      — Доспех
artifact   — Артефакт
wondrous   — Чудесный предмет
potion     — Зелье
scroll     — Свиток
```

⚠️ Если написать значение, которого нет в списке (например `rarity: "крутой"`),
сайт не сломается, просто фильтр по редкости/типу не сможет найти этот предмет —
он не будет пропадать с сайта, но и не будет правильно сортироваться.

---

## Шаг 3. Добавь картинку (если есть)

1. Картинки лежат в папке `images/` в корне репозитория.
2. Зайди в эту папку на GitHub → **"Add file" → "Upload files"** → перетащи
   картинку → **Commit changes**.
3. Имя файла лучше тоже писать латиницей без пробелов, например
   `krasnyy-vihr.jpg` или `brooch-everbloom.png` — некоторые системы плохо
   дружат с кириллицей и пробелами в именах файлов.
4. В `items.json` в поле `"image"` укажи путь **от корня сайта**:
   `"images/krasnyy-vihr.jpg"`.

Если картинки пока нет — не страшно, просто оставь `"image": ""`.

---

## Шаг 4. Допиши новую карточку в items.json

Важно — JSON очень строг к запятым. Перед новым предметом нужно поставить
запятую после закрывающей `}` предыдущего:

```json
[
  {
    "id": "krasnyy-vihr",
    "name": "Красный Вихрь",
    ...
    "description": "..."
  },
  {
    "id": "brooch-everbloom",
    "name": "Брошь Вечного Цветения",
    ...
    "description": "..."
  },
  {
    "id": "novyy-predmet",
    "name": "Твой новый предмет",
    "rarity": "legendary",
    "type": "wondrous",
    "subtype": "Кольцо",
    "image": "images/novyy-predmet.jpg",
    "description": "Описание предмета..."
  }
]
```

**Правила, которые легко нарушить случайно:**

- После **последнего** предмета в списке запятой быть не должно.
- После остальных предметов запятая **обязательна**.
- Кавычки — только обычные прямые `"..."`, не «ёлочки» и не curly-кавычки
  (если копируешь текст из Word — это частая причина поломки).
- Если в самом описании встречается кавычка `"` — перед ней нужно поставить
  обратный слэш: `\"`, иначе JSON сломается.
- Перенос строки внутри `description` не нужен — пиши описание одной строкой,
  сайт сам красиво перенесёт текст на странице.

---

## Шаг 5. Проверь, что не сломал JSON

Перед тем как сохранять — особенно если правишь вручную и не уверен в
запятых — вставь весь файл целиком сюда: **https://jsonlint.com** и нажми
"Validate JSON". Если он ругается красным — он покажет строку с ошибкой.
Если зелёный "Valid JSON" — можно сохранять спокойно.

---

## Шаг 6. Сохрани изменения

Внизу страницы редактора GitHub:

1. В поле commit message коротко напиши что изменил, например:
   `Добавлен предмет: Кольцо Зимней Стужи`.
2. Выбери **"Commit directly to the main branch"**.
3. Нажми зелёную кнопку **"Commit changes"**.

GitHub Pages пересоберёт сайт автоматически — обнови страницу сайта
через 30–60 секунд и новый предмет должен появиться в галерее.

---

## Если что-то не отображается

- **Предмет не появился вообще** → скорее всего сломан JSON (пропущена/лишняя
  запятая или кавычка). Проверь через jsonlint.com.
- **Вместо картинки золотая иконка** → путь в `image` неверный, либо файл не
  загружен в `images/`, либо опечатка в имени файла (регистр важен:
  `Sword.jpg` ≠ `sword.jpg`).
- **Предмет не находится через фильтр редкости/типа** → проверь, что
  `rarity`/`type` написаны ровно так, как в таблицах выше (английскими
  словами, без опечаток, в нижнем регистре).

---

## Шаблон для копирования

Можно просто копировать этот кусок и менять значения под новый предмет:

```json
  {
    "id": "",
    "name": "",
    "rarity": "common",
    "type": "wondrous",
    "subtype": "",
    "image": "",
    "description": ""
  }
```
